# 盼达 Panda｜自由加载中

盼达是运行在飞书群聊中的事件驱动型下班氛围机器人。它观察真实工作收尾信号，在下午不可预测地偶尔出现，为团队制造一点接近自由的期待感。

> 有盼头，才有劲头。

## 核心行为

- 上午 `08:35–11:30` 和下午 `12:45–17:30` 处理工作事件。
- 上午的收尾能量经过时间衰减后影响下午。
- `14:30` 后才可能发言，但没有固定触发时间。
- `17:30` 后绝对静默。
- 普通消息只触发计算，消息数量不直接增加自由能量。
- 真实的完成、跑通、交付、发布和整体收尾信号增加能量。
- 日常吐槽不压制机器人，明确事故会触发临时沉默。
- 每日最多主动出现 4 次，每次随机冷却 30～45 分钟。
- 主动发言可随机选择独立群消息或回复近期收尾消息。
- 每次出现后的 5 分钟内，最多回应一次群成员的回复或 `@`。
- 不评价个人，不做考勤，不保存长期聊天正文。

完整项目约束见 [AGENTS.md](AGENTS.md)，实施范围和验收标准见 [PLAN.md](PLAN.md)。

## 技术架构

```text
飞书长连接事件
      │
      ▼
事件标准化与幂等去重
      │
      ▼
短期内存上下文 ──► 规则分类器
      │                 │
      └────────┬────────┘
               ▼
       自由能量规则引擎
               │
       ┌───────┴────────┐
       ▼                ▼
   影子模式          正式发送
       │                │
       └───────┬────────┘
               ▼
      SQLite 派生状态与匿名反馈
```

核心规则不依赖飞书、数据库或网络，可通过固定时钟和随机种子复现。

## 飞书应用配置

1. 在飞书开放平台创建企业自建应用并启用机器人能力。
2. 将机器人可用范围设置为测试群中的成员，并把机器人加入目标群。
3. 在“事件与回调”中选择“使用长连接接收事件”。
4. 订阅以下事件：
   - 接收消息 `im.message.receive_v1`
   - 添加表情回应 `im.message.reaction.created_v1`
   - 删除表情回应 `im.message.reaction.deleted_v1`
5. 开通接收群消息、以应用身份发送消息、读取表情回应所需权限。
6. 发布应用版本，使新增权限和事件订阅生效。

长连接只要求运行环境能够访问公网，不需要公网 IP、域名或内网穿透。

## 本地运行

项目要求 Python 3.12～3.14，推荐使用 uv 管理环境。

```bash
uv sync --all-groups
cp .env.example .env
```

编辑 `.env`：

```dotenv
LARK_APP_ID=cli_xxx
LARK_APP_SECRET=your_app_secret
PANDA_TARGET_CHAT_ID=oc_xxx
PANDA_PRIVACY_SALT=replace_with_a_random_local_value
PANDA_MODE=shadow
```

启动：

```bash
uv run --env-file .env panda-bot
```

首次运行使用 `shadow` 模式。该模式正常接收、分类、抽签和更新状态，只在日志中记录“本应发送”的文案编号，不向群内发送消息。观察 3～5 个工作日并校准 `config/rules.yaml` 后，将模式切换为：

```dotenv
PANDA_MODE=live
```

正式模式首次启动时会发送一次欢迎语。

## Docker 运行

```bash
docker compose build
docker compose up -d
docker compose logs -f panda-bot
```

容器使用非 root 用户运行，SQLite 数据保存在 `panda-data` 持久化卷中。更新容器不会删除群状态。

停止服务：

```bash
docker compose down
```

## 配置说明

所有业务参数位于 `config/rules.yaml`：

- 工作时段和最早发言时间。
- 活跃对话轮次和参与人数。
- 短期上下文范围。
- 初始门槛、触发后能量保留和门槛增量。
- 能量衰减、概率阶梯、冷却和每日上限。
- 收尾短语、否定短语、风险短语和日常吐槽。
- 独立消息与回复消息的随机比例。

所有文案位于 `config/messages.yaml`。文案编号必须全局唯一，配置不合法时程序会在启动阶段直接报错。

## 数据与隐私

SQLite 仅保存：

- 群标识、日期、能量、门槛、冷却和触发次数。
- 使用项目盐值单向哈希后的活跃成员集合与合并后的对话轮次。
- 近期文案编号、幂等事件 ID 和配置版本。
- 文案主题、发送形式、表情总数及短期互动结果。

最近 10 分钟内最多 5 条同发送者文字消息可用于拆句理解，正文只存在于进程内存中，最长保留 30 分钟。消息正文、图片、文件、个人贡献和回应者身份不会写入数据库或普通日志。

## AI 接入计划

第一阶段运行不依赖 AI。后续能力通过适配器接入：

1. 仅对规则无法确定的候选消息执行结构化语义分类。
2. 规则引擎确认允许发送后，AI 偶尔生成现场文案。
3. 根据匿名表情数量和短期互动结果调整文案主题权重。

外部 AI 最多接收当前候选消息和两条必要上文。姓名、`@`、链接和可识别编号在调用前删除。AI 不决定能量、概率、冷却、每日上限和最终发送行为。

## 日志与排查

关键流程使用标准日志输出，不记录消息正文。常见启动问题：

- 缺少环境变量：检查 `.env` 中的应用凭证和目标群 ID，并使用 `uv run --env-file .env panda-bot` 启动。
- 长连接失败：确认应用为企业自建应用、事件订阅方式是长连接且应用能够访问公网。
- 收不到群消息：确认机器人已加入群、权限已开通并已发布新版本。
- 无实际发言：确认 `PANDA_MODE=live`，当前时间位于允许区间，并检查日志中的决策原因。
