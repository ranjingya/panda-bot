# 盼达规则与决策原因

本文档集中说明盼达纯规则阶段的事件过滤、分类顺序、能量规则、时间兜底、发送限制和日志 `reason`。运行参数以 [`config/rules.yaml`](../config/rules.yaml) 为唯一配置真源，本文档描述当前配置对应的实际行为。

## 判定总流程

每条飞书事件依次经过以下步骤：

1. 适配器排除机器人、应用、系统消息和非文字消息。
2. 服务排除非目标群、空文本和已经消费的重复事件。
3. `@盼达 /status` 优先进入状态命令，不累计活跃度，不改变能量、冷却或互动次数。
4. 机器人主动出现后的 5 分钟内，回复机器人消息或普通 `@盼达` 最多触发一次回嘴。
5. 合并同一成员 60 秒内的连续拆句，并更新匿名活跃人数与有效对话轮次。
6. 分类器按照风险、未完成、忽略表达、收尾信号、短期上下文、日常吐槽、普通消息的顺序分类。
7. 收尾信号优先进入能量与门槛抽签；只有普通消息才可能进入低优先级时间兜底。
8. 真实发送前再次检查时间、每日上限、冷却、风险静默和活跃条件。

机器人没有定时唤醒任务。所有判断都由新事件驱动，没有新消息时不会自行发送。

## 工作时段与活跃条件

| 项目 | 当前值 | 行为 |
|---|---:|---|
| 时区 | `Asia/Shanghai` | 所有工作时段和日期边界使用上海时间 |
| 上午工作时段 | `08:35–11:30` | 收尾信号可积累能量，但不得主动发言 |
| 下午工作时段 | `12:45–17:30` | 更新活跃度并执行规则判断 |
| 最早发言时间 | `14:30` | 时间边界，不是定时触发点 |
| 绝对静默时间 | `17:30` | 主动发言、回嘴和状态命令全部静默 |
| 下午最少参与人数 | 2 人 | 使用匿名成员标识统计 |
| 下午最少有效轮次 | 12 轮 | 同一成员 60 秒内连续拆句合并为一轮 |
| 短期上下文 | 10 分钟内最多 5 条 | 只在内存中辅助拆句理解 |
| 内存上下文最长保留 | 30 分钟 | 仅辅助当前规则判断 |
| Shadow 脱敏正文保留 | 5 天 | 每次采集自动清理，Live 不保存正文 |

周末和节假日没有单独日历判断，仍以群聊当天的实际参与人数和有效轮次决定是否具备触发资格。

## 消息分类规则

分类器按照固定优先级处理文本，前面的分类一旦命中就不再继续向后匹配。

### 风险信号

风险优先级最高。当前包括线上或生产故障、数据异常或丢失、客户投诉、紧急修复、回滚、业务影响、全站或服务不可用和严重事故。命中后开启 45 分钟保护静默，不增加能量，也不参与时间兜底。

### 未完成表达

包括还没完成、完成不了、没搞定、没做完、没改好、没跑通、仍有 bug、持续报错、准备返工以及发布、上线或交付失败。此类消息不增加能量，也不参与时间兜底。

### 忽略表达

包括想下班、提前下班、可以下班、下班再处理、不管了、就这样吧和准备好了。这些表达不提供能量或召唤能力，也不参与时间兜底。

### 收尾信号与能量

| 信号类 | 当前能量范围 | 典型表达 |
|---|---:|---|
| `overall_finish` | 20～30 | 全部结束、全部搞定、今天收尾、收尾完成、完美收官 |
| `delivery` | 18～25 | 已经发布、发布成功、已上线、交付完成、进入待交付、已经发出 |
| `verified` | 15～22 | 测试或验收通过、客户确认、跑通了、基本稳定、稳定运行 |
| `general_finish` | 12～18 | 完成了、搞定了、处理好了、做完了、写完了、改好了、修完了、结束了 |

“好了、成了、完了、结束了、完成了、搞定了、稳定了、跑通了、通过了”等短句可以结合同一成员近期上下文判断。组合结果仍必须命中收尾词表，并且不能包含未完成表达。

### 日常吐槽与普通消息

烦死了、好烦、服了、难绷、要命、崩溃、受不了、不想干和救命属于日常吐槽。吐槽不增加能量、不启动风险静默，也不参与时间兜底。其他未命中规则的文本属于普通消息，可以更新活跃度并参与时间兜底。

## 能量、门槛与收尾抽签

每天首次事件会创建独立群状态：自由能量从 0 开始，初始门槛在 45～80 之间随机生成。

只有收尾信号增加能量。每次工作事件到达时，会根据距离上一个工作事件的间隔衰减已有能量：

| 事件间隔 | 能量保留比例 |
|---|---:|
| 不超过 30 分钟 | 100% |
| 超过 30 分钟且不超过 90 分钟 | 80% |
| 超过 90 分钟 | 50% |

能量低于门槛时不抽签。达到门槛后，根据超过门槛的数值选择概率：

| 超出门槛 | 单次抽签概率 |
|---|---:|
| 0～9.99 | 20% |
| 10～19.99 | 35% |
| 20～34.99 | 50% |
| 35 及以上 | 75% |

概率永远不会达到 100%。收尾信号触发成功后：

- 只保留触发前能量的 15%～35%。
- 下一次收尾触发门槛增加 12～25。
- 进入 30～45 分钟随机冷却。
- 独立群消息概率为 60%，回复当前收尾消息概率为 40%。

## 低优先级时间兜底

时间兜底仅在当前消息被分类为普通消息时评估。风险、未完成、忽略表达、收尾信号和日常吐槽都不会进入时间兜底。

时间兜底还必须同时满足：

- 当前消息构成新的有效对话轮次。
- 当前时间位于 `14:30–17:30`。
- 下午至少 2 人参与并形成至少 12 个有效轮次。
- 未达到每日主动发言总上限。
- 不在冷却或风险静默期间。
- 当天尚未使用时间兜底。

| 生效时间 | 单次普通轮次概率 |
|---|---:|
| `14:30` 起 | 1% |
| `15:30` 起 | 2% |
| `16:30` 起 | 4% |
| `17:00` 起 | 8% |

时间兜底每天最多成功 1 次，计入每天最多 4 次主动发言的总上限。成功后始终发送独立群消息，并进入相同的 30～45 分钟冷却，但不会增加、消耗或重置能量和门槛。

## 互动与状态命令

每次主动发言后开启 5 分钟互动窗口。在窗口内回复机器人消息或 `@盼达`，最多随机回嘴一次。回嘴不增加能量、不重置冷却，也不计入主动发言次数；`17:30` 后不回嘴。

只有 `@盼达 /status` 是有效状态命令。单独发送 `/status` 会被静默忽略，并且不累计活跃度、不进入能量或时间兜底判断。

正式模式下的状态命令展示当前运行阶段、能量、动态门槛、当前概率档位、冷却剩余时间、保护静默、匿名活跃人数与轮次、今日冒泡及时间兜底次数和规则版本。状态命令不展示关键词、完整概率阶梯、门槛范围、冷却范围、每日上限、成员身份或个人数据。影子模式和工作时间外不回复状态命令。

## 运行模式

### 影子模式 `shadow`

影子模式完整执行分类、能量、抽签、冷却和状态提交，但不向群内发送任何消息。首次欢迎语、主动消息、回嘴和状态命令全部静默。

目标群内通过前置过滤的成员文字都会形成一条 Shadow 观察，包括未命中任何预设短语的普通消息。观察保存脱敏正文、每天变化的匿名成员别名、是否为新轮次、分类结果、决策原因、能量、门槛、概率、抽签值、匿名活跃度以及“本应发送”的文案编号。默认保留 5 天，每次写入时自动清理过期数据；使用 `panda-shadow-export` 可以导出按时间排列的 AI 校准报告。

### 正式模式 `live`

正式模式会在规则允许时真实发送消息。首次进入目标群发送一次欢迎语，之后所有主动发言和互动遵守时间、风险、冷却与上限限制。

## 分类原因

分类原因描述当前文本被识别成什么。

| reason | 含义 | 后续行为 |
|---|---|---|
| `empty_text` | 规范化后文本为空 | 不增加能量，不触发 |
| `explicit_risk` | 命中明确风险词 | 开启保护静默 |
| `negative_or_unfinished` | 命中未完成或失败表达 | 不增加能量，不参与时间兜底 |
| `ignored_expression` | 命中忽略表达 | 不增加能量，不参与时间兜底 |
| `completion_pattern` | 当前消息直接命中收尾信号 | 增加能量并继续收尾触发判断 |
| `completion_from_short_context` | 短句结合近期上下文命中收尾信号 | 增加能量并继续收尾触发判断 |
| `casual_complaint` | 命中日常吐槽 | 不增加能量，不参与时间兜底 |
| `ordinary_message` | 未命中其他类别的普通消息 | 更新活跃度并可评估时间兜底 |

## 触发决策原因

触发决策原因是日志中 `reason=...` 的主要排查依据，表示本次事件为什么没有发送或通过了哪条触发路线。

| reason | 含义 | 是否可能改变状态 |
|---|---|---|
| `outside_work_time` | 事件不在上午或下午工作时段 | 否 |
| `risk_suppression_started` | 当前风险消息启动保护静默 | 设置 `risk_until` |
| `outside_send_window` | 当前时间不在 `14:30–17:30` | 可保留此前新增的能量与活跃度 |
| `daily_limit_reached` | 当天已经主动发言 4 次 | 可保留当前新增的收尾能量 |
| `cooldown_active` | 距离上次主动发言仍在冷却期 | 可保留当前新增的收尾能量 |
| `risk_suppression_active` | 当前仍处于保护静默期 | 可保留当前新增的收尾能量 |
| `not_enough_active_senders` | 下午真实参与人数少于 2 人 | 可更新活跃度并保留收尾能量 |
| `not_enough_active_turns` | 下午有效轮次少于 12 轮 | 可更新活跃度并保留收尾能量 |
| `energy_below_threshold` | 收尾能量仍低于当前随机门槛 | 保留新增后的能量 |
| `probability_missed` | 能量达到门槛，但本次收尾抽签失败 | 保留当前能量，不进入冷却 |
| `triggered` | 收尾信号通过全部条件和抽签 | 影子提交或正式发送 |
| `time_fallback_requires_new_turn` | 普通消息被合并进已有对话轮次 | 不执行时间兜底抽签 |
| `time_fallback_daily_limit_reached` | 当天已经成功使用一次时间兜底 | 不改变能量 |
| `time_fallback_not_started` | 当前时间早于首个时间兜底区间 | 不改变能量 |
| `time_fallback_probability_missed` | 时间兜底单次抽签失败 | 不改变能量，不进入冷却 |
| `time_fallback_triggered` | 普通消息通过时间兜底抽签 | 影子提交或正式独立发送 |

分类为未完成、忽略表达、日常吐槽或普通消息但不适用时间兜底时，决策原因可能直接沿用对应的分类 `reason`。

## 没有 reason 的前置过滤

以下事件在进入核心引擎前直接忽略，因此不会产生上述决策原因：

- 非目标群消息。
- 机器人、应用或系统发送的消息。
- 图片、文件、贴纸、转发等非文字消息。
- 空白正文。
- 已经消费过的重复事件。
- `@盼达 /status` 和单独发送的 `/status` 不进入能量与触发引擎；Shadow 模式仍以 `status_command` 保存脱敏观察。

重复事件会记录“忽略重复事件”日志；其他前置过滤默认静默，以避免输出消息正文、成员身份或无意义噪声。

## 持久化状态

SQLite 的 `group_states.state_json` 保存当前群的日期、能量、动态门槛、主动发言次数、时间兜底次数、冷却和风险截止时间、匿名活跃人数与轮次、近期文案编号、互动窗口和配置版本。

`feedback` 保存每次影子命中或正式发送的文案编号、主题、发送形式、发送时间、匿名表情总数、是否收到回复、是否回嘴以及是否属于影子记录。

`shadow_observations` 仅在 Shadow 模式写入目标群成员的脱敏文字与规则决策快照。正文会删除 `@`、链接、邮箱、飞书标识和长编号，成员别名只在同一天稳定，默认保留 5 天。Live 模式不写入该表。数据库不保存原始成员 ID、个人贡献分数和回应者身份。
