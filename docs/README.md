# 盼达 Panda｜自由加载中

盼达是运行在飞书群聊中的事件驱动型下班氛围机器人。它优先观察真实工作收尾信号，也会在活跃下午使用低概率时间兜底，在不可预测的时刻偶尔出现，为团队制造一点接近自由的期待感。

> 有盼头，才有劲头。

## 核心行为

- 上午 `08:35–11:30` 和下午 `12:45–17:30` 处理工作事件。
- 上午的收尾能量经过时间衰减后影响下午。
- `14:30` 后才可能发言，但没有固定触发时间。
- `17:30` 后绝对静默。
- 普通消息只触发计算，消息数量不直接增加自由能量。
- 真实的完成、跑通、交付、发布和整体收尾信号增加能量。
- 普通消息未命中收尾信号时，活跃群可按时间概率低优先级兜底；兜底每天最多一次且只发送独立群消息。
- 日常吐槽不压制机器人，明确事故会触发临时沉默。
- 每日最多主动出现 4 次，每次随机冷却 30～45 分钟。
- 收尾信号触发时可随机选择独立群消息或回复近期收尾消息。
- 每次出现后的 5 分钟内，最多回应一次群成员的回复或 `@`。
- 工作时段内发送 `@盼达 /status` 可查看动态运行状态，不影响能量、冷却和互动次数。
- 不评价个人，不做考勤；正式模式不持久化正文，影子模式只短期保存脱敏校准语料。

完整项目约束见 [AGENTS.md](../AGENTS.md)，实施范围和验收标准见 [PLAN.md](PLAN.md)，规则参数与日志原因见 [规则与决策原因](RULES_AND_REASONS.md)。

## 技术架构

```text
飞书长连接事件
      │
      ▼
事件标准化与幂等去重
      │
      ▼
短期内存上下文 ──► 规则分类器
      │                 │
      └────────┬────────┘
               ▼
       自由能量规则引擎
        │      │
    收尾优先  时间兜底
        │      │
        └──┬───┘
           │
       ┌───┴────────────┐
       ▼                ▼
   影子模式          正式发送
       │                │
       └───────┬────────┘
               ▼
      SQLite 派生状态、Shadow 校准语料与匿名反馈
```

核心规则不依赖飞书、数据库或网络，可通过固定时钟和随机种子复现。

## 飞书应用配置

1. 在飞书开放平台创建企业自建应用并启用机器人能力。
2. 将机器人可用范围设置为测试群中的成员，并把机器人加入目标群。
3. 在“事件与回调”中选择“使用长连接接收事件”。
4. 订阅以下事件：
   - 接收消息 `im.message.receive_v1`
   - 添加表情回应 `im.message.reaction.created_v1`
   - 删除表情回应 `im.message.reaction.deleted_v1`
5. 开通接收群消息、以应用身份发送消息、读取表情回应所需权限。
6. 发布应用版本，使新增权限和事件订阅生效。

长连接只要求运行环境能够访问公网，不需要公网 IP、域名或内网穿透。

## 本地运行

项目要求 Python 3.12～3.14，推荐使用 uv 管理环境。

```bash
uv sync --all-groups
cp .env.example .env
```

编辑 `.env`：

```dotenv
LARK_APP_ID=cli_xxx
LARK_APP_SECRET=your_app_secret
PANDA_TARGET_CHAT_ID=oc_xxx
PANDA_PRIVACY_SALT=replace_with_a_random_local_value
PANDA_MODE=shadow
```

启动：

```bash
uv run --env-file .env panda-bot
```

首次运行使用 `shadow` 模式。该模式正常接收、分类、抽签和更新状态，不向群内发送任何消息；同时保存目标群有效成员文字的脱敏正文及当时的分类、决策、能量和活跃度快照。默认保留 5 天，每次采集时自动删除过期记录。

运行 3～5 个工作日后导出 AI 可读的校准报告：

```bash
uv run --env-file .env panda-shadow-export --days 5 --output data/shadow-review.md
```

导出文件位于已被 Git 忽略的 `data/` 目录。可以让 AI 读取该报告，找出真实收尾表达的漏判与误判，并总结群聊节奏、常用梗和文案风格建议。完成校准并更新 `config/rules.yaml`、`config/messages.yaml` 后，将模式切换为：

```dotenv
PANDA_MODE=live
```

正式模式首次启动时会发送一次欢迎语。

## Docker 运行

```bash
docker compose build
docker compose up -d
docker compose logs -f panda-bot
```

容器使用非 root 用户运行，SQLite 数据保存在 `panda-data` 持久化卷中。更新容器不会删除群状态。

停止服务：

```bash
docker compose down
```

## 配置说明

所有业务参数位于 `config/rules.yaml`：

- 工作时段和最早发言时间。
- 活跃对话轮次和参与人数。
- 短期上下文范围。
- 初始门槛、触发后能量保留和门槛增量。
- 能量衰减、收尾概率阶梯、时间兜底概率、冷却和每日上限。
- 收尾短语、否定短语、风险短语和日常吐槽。
- 独立消息与回复消息的随机比例。

所有文案位于 `config/messages.yaml`。文案编号必须全局唯一，配置不合法时程序会在启动阶段直接报错。

## 群内交互

- 机器人主动出现后的 5 分钟内，回复机器人消息或 `@盼达` 最多触发一次随机回嘴。
- 互动窗口外的普通 `@盼达` 不提供强制召唤能力，消息仍按正常规则判断。
- 只有 `@盼达 /status` 会回复当前运行阶段、能量、动态门槛、当前概率档位、冷却、保护静默、匿名活跃度、今日冒泡次数和规则版本。
- 单独发送 `/status` 时保持静默，且不进入活跃度、能量和时间兜底判断。
- 状态命令不公开关键词、概率阶梯、门槛范围、冷却范围、每日上限和个人信息。
- 影子模式、工作时间外以及 `17:30` 后收到状态命令时保持静默。

## 数据与隐私

SQLite 保存：

- 群标识、日期、能量、门槛、冷却、触发次数和时间兜底次数。
- 使用项目盐值单向哈希后的活跃成员集合与合并后的对话轮次。
- 近期文案编号、幂等事件 ID 和配置版本。
- 文案主题、发送形式、表情总数及短期互动结果。
- 仅在 Shadow 模式下保存脱敏成员文字和每条消息的完整规则决策快照。

最近 10 分钟内最多 5 条同发送者文字消息可用于拆句理解，内存上下文最长保留 30 分钟。Shadow 语料会删除 `@`、链接、邮箱、飞书标识和长编号，成员使用每天变化的匿名别名，默认保留 5 天；Live 模式不写入正文。图片、文件、贴纸、转发、个人贡献、长期成员标识和回应者身份不进入校准语料，普通日志始终不输出消息正文。

## AI 接入计划

第一阶段运行不依赖 AI。后续能力通过适配器接入：

1. 使用 Shadow 脱敏校准记录分析漏判、误判和群聊调性，建议更新外部规则与固定文案。
2. 仅对规则无法确定的候选消息执行结构化语义分类。
3. 规则引擎确认允许发送后，AI 偶尔生成现场文案。
4. 根据匿名表情数量和短期互动结果调整文案主题权重。

运行时外部 AI 最多接收当前候选消息和两条必要上文。成员 `@`、链接、邮箱、飞书标识和可识别长编号在调用前删除。AI 不决定能量、概率、冷却、每日上限和最终发送行为。

## 日志与排查

关键流程使用标准日志输出，不记录消息正文。常见启动问题：

- 缺少环境变量：检查 `.env` 中的应用凭证和目标群 ID，并使用 `uv run --env-file .env panda-bot` 启动。
- 长连接失败：确认应用为企业自建应用、事件订阅方式是长连接且应用能够访问公网。
- 收不到群消息：确认机器人已加入群、权限已开通并已发布新版本。
- 无实际发言：确认 `PANDA_MODE=live`，当前时间位于允许区间，并检查日志中的决策原因。
