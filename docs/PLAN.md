# 盼达 Panda 项目实施计划

## 目标

交付一个可在飞书测试群运行的事件驱动型下班氛围机器人。项目使用 Python、uv、SQLite 和飞书长连接，支持影子模式与正式发送模式，并通过 Docker 统一交付。

## 完成范围

- 接收并标准化飞书文字消息事件。
- 执行工作时段、活跃度、自由能量、低优先级时间兜底、衰减、随机门槛、概率抽签、冷却和每日上限规则。
- 识别明确收尾、否定表达、日常吐槽和严重风险信号。
- 合并连续拆句，使用受限的短期内存上下文辅助判断。
- 在独立群消息和回复消息之间随机选择发送形式。
- 支持主动发言后的限次回嘴互动。
- 支持不改变能量、冷却和互动次数的 `@盼达 /status` 动态状态命令。
- 使用 SQLite 保存派生状态、幂等记录和匿名互动效果。
- 提供固定审核文案库、外部规则配置和配置校验。
- 提供结构化日志、影子模式、Docker 镜像和 Compose 配置。
- 提供自动化测试、示例环境变量和运行说明。
- 预留 AI 分类、AI 文案和匿名权重学习接口，第一阶段运行不依赖 AI。

## 实施阶段

### 1. 工程基础

- 调整 `pyproject.toml`，声明运行依赖、开发依赖和命令入口。
- 建立 `src/panda_bot`、`config` 和 `tests` 目录结构。
- 定义配置模型、领域事件、群状态和决策结果。

验收标准：项目能够通过 uv 安装，配置能够加载并对非法参数快速报错。

### 2. 纯规则核心

- 实现消息标准化、拆句合并和短期上下文缓存。
- 实现收尾、否定、吐槽和风险分类。
- 实现跨午休能量衰减、活跃条件、隐藏门槛和概率抽签。
- 实现仅由新对话事件驱动的时间兜底，确保收尾信号优先且普通消息不增加能量。
- 实现每日最多 4 次、30～45 分钟冷却和触发后状态更新。
- 实现文案近期去重、主题轮换和发送形式随机选择。
- 实现状态命令优先路由、静默时段限制、动态状态快照和内置规则字段黑名单。

验收标准：固定时钟和固定随机种子能够完全复现决策结果，核心模块不依赖飞书、SQLite 和网络。

### 3. 持久化与应用编排

- 实现 SQLite 仓储、事务更新和数据库初始化。
- 实现事件幂等、过期记录清理和匿名互动效果记录。
- 实现影子模式与正式模式编排。
- 实现发送前的时间、冷却、上限和风险二次检查。

验收标准：进程重启后状态可恢复，重复事件不会产生重复发送。

### 4. 飞书接入

- 使用飞书官方 Python SDK 建立长连接。
- 解析文字消息、回复、`@盼达` 和表情事件。
- 以应用身份发送独立消息或回复消息。
- 排除机器人自身、其他机器人、非目标群和非文字事件。

验收标准：使用模拟适配器完成端到端测试；填写飞书凭证和目标群 ID 后能够启动真实长连接。

### 5. 运行交付

- 提供审核文案库和默认规则配置。
- 提供 `.env.example`、`Dockerfile`、`compose.yaml` 和 `.dockerignore`。
- 使用非 root 用户运行容器，挂载 SQLite 数据目录，配置健康检查和自动重启。
- 更新 `docs/README.md`，说明飞书配置、影子运行、正式运行、测试和故障排查。

验收标准：本地可通过 `uv run --env-file .env panda-bot` 启动，Docker 配置可解析，镜像具备确定的启动入口。

### 6. 测试与验收

- 覆盖时段边界、午休衰减、拆句、否定、风险、群体条件、频率、冷却、文案、互动、隐私、幂等和降级。
- 执行格式检查、静态检查和完整测试。
- 检查仓库中不存在凭证、消息正文、临时文件和调试输出。

验收标准：全部自动化测试通过，关键分支具备可复现测试，项目文档与实现一致。

## 外部上线条件

真实群内运行需要以下外部配置：

- 飞书自建应用的 App ID 与 App Secret。
- 已启用机器人能力并发布应用版本。
- 应用具备接收群消息、读取群信息、发送消息和读取表情回应所需权限。
- 机器人已加入目标群，并取得目标群 Chat ID。

这些值仅通过运行环境注入，不写入源码、镜像或版本库。
